Important Concepts

Lesson 1: HTML Basics

World Wide Web

The Web is a collection of HTML documents connected through several components. The major components are: Browsers, Servers, Internet, and HTML documents. Browsers, such as Internet Explorer, Firefox, Chrome, make requests to servers through the Internet using HTTP protocol in order to view HTML documents. Servers are computers specialized to host HTML files. Servers will send the HTML document to the Brower, which is then interpretted by the browser to provide a web page for the user.

HTML

HTML stands for Hypertext Markup Language. It's components are text content, markup (or formatting), and references to other documents, e.g. images and videos

HTML Tags and Elements

HTML elements are used to engineer HTML documents. Each element begins with an opening tag and most end with a closing tag. This video explains the structure of elements very thoroughly!

Inline and Block Elements

Block elements draw an invisible container around their text. Anything outside of the block will go to the next line or previous lines. Inline elements can be thought of as characters, in that they will output on the same line as the last text/inline element provided there is room in the line.

Some examples of block elements include <p>, <div>, and <form>.
Some examples of inline elements are formatting elements, such as <b> and <em>.

HTML Document Structure

Each HTML document has a common structure. This video provides a thorough breakdown of the structure of HTML documents. The components of the HTML structure are:

  1. Doctype - Declares version of HTML of document. html as the doctype is HTML5
  2. HTML - element that encloses everything except doctype
  3. Head - Metedata that is not visible in the window, but is relevant to the page goes here
  4. Title - This is the text that is displayed as the title of the window for this page.
  5. Body - This is where the text displayed to the user is placed.

Lesson 2: Creating a Structured Document

Languages Used to Build a Webpage

Three languages used to design web pages are HTML, CSS, and JavaScript. With a web page open in your browser, you can right click anywhere on the web page and select Inspect Element to bring up the developer tools. This allows you to view and edit the brower's interpretation of the page, which includes its interpretation of HTML and CSS code used to create the web page. Each of these languages serve a unique purpose within Front-End Web Development, succinctly described below:

  1. HTML - Provides the page with structure and text.
  2. CSS - Used to style HTML texts and objects, such as font size and color to name a few properties.
  3. JavaScript - Allows the user to interact with the page.

Structure Basics

Elements, enclosed by tags in HTML, become rectangular objects when the page is interpreted by a web browser. These objects form the Document Object Model, or DOM, which is a hierachical tree of the objects on the page. All objects, even ones that are explicitly circles, are technically rectangular objects on the web page. Just right click and press Inspect Element . Additionally, boxifying is the term used to describe visually dividing a web page into its most basic rectangular objects.

Lesson 3: Adding CSS Style

CSS Basics

CSS is the web development language used to stylize web pages. It does this by applying a defined style to elements identified by either a class or tag. CSS stands for Cascading Style Sheets. The term "cascading" arises from the fact that style rules are applied to child elements, a feature that can save developers time. Another great feature is that the child elements styling can be overwritten, if needed, as the most specific style is applied in CSS. When a browswer displays a page to the user, there is an order (think order of operations in algebra), that style is applied. First, the default stayle of the browser's style is applied, then the style specified by a separate .css file overwrites any browser style. Following that, a stylesheet inside of the HTML file is applied after both of the previous options. Finally, any inline style elements are applied last. Ideally, we should only use a separate .css file, but small projects benefit from using a stylesheet internal to the HTML file. Last, but not least, any inline styles are displayed last, however, inline style elements are very cluttering, so they should be avoided.
